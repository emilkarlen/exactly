An actor that does nothing, and that accept an empty act phase.

Usefull when the thing to check exists beforehand - is "global" files
and directories.

* Syntax

** actor intruction

[conf]

actor --null

** command line option

--null-actor

** implicit

The null actor may be set automatically if there is no act phase contents

* Example usages

** At ETI, to check that all SQL scripts have valid names:

----------------------------------------

[conf]

act-home <where sql files are locaated>

[assert]

dir-contents --rel-act-home --selection ! name --reg-ex PATTERN  empty

dir-contents --rel-act-home --selection ! type file              empty

--------------------

*** alt

--------------------

dir-contents --rel-act-home --selection ! ( type file & name --reg-ex PATTERN ) empty

--------------------

*** alt

--------------------

[setup]

def path          SQL_DIR        = --rel-home ../../next_release

def file-selector VALID_SQL_FILE = type file & name --reg-ex PATTERN

[assert]

dir-contents @[SQL_DIR]@ --selection ! VALID_SQL_FILE [is] empty

** For testing Exactly itself

It would be convenient to not have to specify an actor in some test,
for example, the test that the default predefined symbols are actually
predefined.
